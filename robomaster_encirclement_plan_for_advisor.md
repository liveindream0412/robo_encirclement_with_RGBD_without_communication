# Robomaster 三车围捕项目汇报简版（端到端双分支方案）

## 1. 项目目标

本项目的目标是：

- 使用 3 台 Robomaster 小车
- 围住 1 个静止目标
- 每台车独立决策，不依赖集中式动作分配
- 先在学校 Ubuntu 22.04 环境中的 Isaac Sim 5.1.0 + Isaac Lab 中训练
- 后续迁移到真实机器人系统

当前更关注：

- 项目是否可行
- 技术路线是否合理
- 哪些环节最容易出问题

而不是优先追求论文完整复现。

---

## 2. 当前已知条件

目前已知系统条件较好：

1. **控制与仿真环境已有基础**
   - 仿真平台、控制链路基本现成
   - 不需要从底层控制重新开始搭建

2. **模型输出接口较清晰**
   - 模型最终只需要输出 3 个高层值
   - 这 3 个值可理解为局部参考目标或高层运动参考量
   - 下层现成控制器负责执行

3. **已有感知与状态来源**
   - 可获取深度图
   - 可获取自车绝对位置（动捕）

因此，本项目当前最核心的问题不是底层控制，而是：

> 如何设计一个高层协同策略，把深度图和自车状态结合起来，输出合适的高层参考量，使 3 台车形成稳定围捕。

---

## 3. 核心技术思路

### 3.1 整体思路

根据老师要求，建议采用：

- **端到端高层协同强化学习**
- **低层控制器复用现有系统**
- **集中训练、分散执行（CTDE）**
- **三车共享同一套策略参数**

这里的“端到端”具体指：

- 深度图不再先做人工几何特征提取后再喂给策略
- 而是直接送入策略网络的视觉分支
- 自车状态（动捕位姿、朝向、速度等）直接送入状态分支
- 网络内部完成特征提取与融合
- 最终直接输出 3 个高层参考值

即：

- 训练时可以利用更多全局信息辅助 critic 和 reward 计算
- 执行时每台车只根据自己的深度图和自车状态独立输出动作

---

### 3.2 为什么不是纯几何特征 + RL

原先方案里，基础版更偏“结构化观测优先”，是为了更快验证任务定义和 reward。

但根据老师要求，当前更推荐的主线是：

> **深度图直接进入网络，采用端到端学习；同时保留自车状态分支，使网络在视觉信息之外也能利用稳定的动捕状态。**

这样做的原因是：

1. 更符合老师提出的“直接做端到端”的要求
2. 更符合后续真正的视觉协同任务形态
3. 动捕状态本来就是系统可直接使用的信息，不必人为舍弃
4. 比只靠纯视觉单分支更容易训练和部署

因此，本项目不是“纯视觉单输入端到端”，而是：

> **深度图 + 自车状态的多模态端到端高层协同策略。**

---

## 4. 推荐系统架构

建议系统分为 5 层：

1. **感知输入层**
   - 深度图直接作为视觉输入
   - 不手工提取目标距离、目标角度、队友角度等几何特征后再喂给 actor

2. **状态输入层**
   - 动捕提供自车绝对位置、朝向、速度等稳定状态信息

3. **高层协同策略层**
   - 输入：深度图 + 自车状态
   - 网络内部完成视觉编码、状态编码、融合和时序建模
   - 输出：3 个高层参考量

4. **安全过滤层**
   - 对策略输出做限幅、最小间距保护、紧急停等修正

5. **底层控制层**
   - 由现成控制器执行高层参考命令

该架构的优势是：

- 符合端到端要求
- 不破坏现有控制链路
- 可直接复用动捕信息
- 更适合后续真实部署

---

## 5. 端到端双分支网络方案

这是当前最关键的设计部分。

### 5.1 双分支的含义

所谓“双分支”，是指网络内部有两条输入处理路径：

1. **视觉分支**
   - 输入：深度图
   - 作用：学习目标、队友、障碍物、可通行空间、遮挡关系等局部环境特征

2. **状态分支**
   - 输入：自车状态（如位置、朝向、速度、上一时刻动作等）
   - 作用：提供稳定的自身几何参考和运动状态信息

这两条分支分别提取特征后，再在网络中间进行融合。

最通俗的理解是：

- 深度图分支像“眼睛”
- 状态分支像“我知道自己现在在哪、朝哪、怎么动”
- 融合后的网络像“大脑”
- 现成控制器像“身体执行器”

---

### 5.2 具体怎么融合

推荐采用最稳妥的**中间融合（mid-level fusion）**。

即：

#### 第一步：视觉分支单独编码

深度图经过 CNN 编码器，得到视觉特征：

- `f_depth = CNN(depth_image)`

这一步不要求网络直接理解“目标距离是多少”，而是让网络自己从深度图中学出有用表示。

#### 第二步：状态分支单独编码

自车状态经过 MLP 编码器，得到状态特征：

- `f_ego = MLP(ego_state)`

#### 第三步：特征拼接融合

将两部分特征拼接：

- `f_fuse = concat(f_depth, f_ego)`

然后再经过一个融合 MLP：

- `f_joint = MLP_fuse(f_fuse)`

#### 第四步：时序建模

将融合后的特征送入 GRU：

- `h_t = GRU(f_joint, h_{t-1})`

作用是处理：

- 深度图遮挡
- 短时漏检
- 局部视野不完备
- 围捕动作需要连续性的问题

#### 第五步：动作输出

GRU 输出再经过策略头，得到最终 3 个高层动作值：

- `a_t = policy_head(h_t)`

critic 训练时可额外使用更全局的信息，但 actor 执行时只用本车输入。

---

### 5.3 为什么推荐中间融合，而不是别的融合方式

常见融合方式有三种：

1. **早期融合**
   - 一上来就把图像和状态硬拼
   - 不太适合不同模态，通常不推荐

2. **中间融合**
   - 各自先编码，再拼接融合
   - 最稳妥、最常见、最适合当前项目

3. **后期融合**
   - 各自独立出动作或决策，再融合
   - 对当前任务不够自然

因此当前最推荐：

> **深度图 CNN 编码 + 状态 MLP 编码 + 特征拼接 + 融合 MLP + GRU + 动作头**

这是最清楚、最容易实现、也最容易向老师解释的端到端双分支方案。

## 6. 多智能体训练方式建议

建议采用：

- **参数共享（parameter sharing）**
- **集中训练、分散执行（CTDE）**

理由：

1. 三台 Robomaster 是同构平台
2. 共享策略实现简单、训练效率高
3. 执行时每台车独立决策，更符合真实场景
4. critic 可以在训练时利用更多全局信息提升稳定性

这是当前最适合 3 台同构车围捕任务的多智能体学习范式。

---

## 7. Reward 设计思路

Reward 设计仍然是项目成败关键。

围捕任务不能只奖励“靠近目标”，否则很容易出现三台车全部挤在目标一侧的现象。

建议 reward 至少包含以下几项：

1. **接近目标奖励**
   - 鼓励车辆向目标靠近

2. **环带奖励**
   - 鼓励车辆停在目标周围合适半径上，而不是直接撞向目标

3. **角度均匀奖励**
   - 鼓励三台车围绕目标形成近似 120° 的均匀分布

4. **最大缺口惩罚**
   - 防止目标周围某一侧空得很大

5. **碰撞惩罚**
   - 防止车车碰撞、车目标过近、车障碍碰撞

6. **扎堆惩罚**
   - 防止几台车虽然没撞，但仍然挤得很近

7. **平滑惩罚**
   - 让输出动作更稳定，便于真机执行

8. **时间惩罚**
   - 鼓励更快完成围捕

9. **稳定保持奖励**
   - 鼓励车辆持续保持围捕结构，而不是偶然经过一个好位置

需要强调的是：

- actor 输入采用端到端深度图 + 状态
- 但 reward 仍然可以由仿真真值几何量来计算

这并不冲突，也是强化学习里非常常见的做法。

---

## 8. 围捕成功条件建议

建议 success 不定义为“大家都靠近目标”，而定义为：

1. 三台车都进入目标周围合理半径带
2. 三台车相对目标的角度分布接近 120°
3. 连续保持若干步
4. 无碰撞

这样更符合“真正围住目标”的直觉，也更利于后续定量评测。

---

## 9. 规则 baseline 的定位调整

在老师要求端到端的前提下，规则 baseline 仍然有价值，但定位需要调整。

它不再作为主输入特征来源，而主要作为：

1. **对照组**
   - 用来比较 RL 是否比规则法更好

2. **teacher / warm-start 来源**
   - 如果后续训练不稳定，可利用规则法采数据做行为克隆预热

3. **残差学习基础控制器**
   - 如果纯端到端 RL 太难训，可考虑 residual RL

也就是说：

> 规则 baseline 仍然建议保留，但不应替代端到端视觉策略本身。

---

## 10. 训练路线建议

在当前端到端要求下，建议训练路线如下：

### 路线 A：直接端到端主线

1. 深度图 + 自车状态作为 actor 输入
2. 三车共享 actor
3. 中央 critic 用全局信息训练
4. 直接用 reward 学习端到端围捕策略

这是最符合老师要求的主线。

### 路线 B：如果训练不稳的备选增强

如果直接端到端训练出现：

- 长时间学不会靠近目标
- 三车长期乱撞
- 动作抖动严重
- 收敛极慢

则可引入：

1. 规则 baseline 对照
2. 行为克隆预热
3. residual RL

也就是说：

- 主方向是端到端
- 规则先验是增强手段，而不是主方法

---

## 11. 推荐训练流程

建议按 curriculum 分阶段推进：

### S0：规则 baseline 跑通（可选但推荐）
- 先有一个非学习围捕器形成基本包围
- 用于验证场景和指标是否合理

### S1：端到端单车接近静止目标
- 深度图 + 自车状态输入
- 学会基本靠近和进入环带

### S2：端到端双车接近 + 避碰
- 学会不抢位置、不撞队友

### S3：端到端三车完整围捕
- 学会形成完整围捕结构

### S4：加入感知噪声与遮挡
- 增强 sim-to-real 鲁棒性

### S5：加入控制扰动与动力学随机化
- 进一步适应真实执行偏差

## 12. 预期主要技术难点

建议重点请老师帮忙评估以下技术难点：

### 12.1 模型输出 3 个值的具体语义
当前只知道最终输出为 3 个高层值，但还需要进一步明确：

- 是局部位移目标？
- 是参考点坐标？
- 是速度类参考量？

这会影响动作空间定义和 reward 平滑项设计。

### 12.2 深度图输入形式如何处理
需要明确：

- 输入深度图分辨率多大合适
- 是否做裁剪、降采样
- 是否输入单帧还是短序列
- 是否需要堆叠若干帧

### 12.3 端到端网络规模如何控制
需要平衡：

- 视觉分支表达能力
- 训练稳定性
- Isaac Lab 并行训练吞吐
- 后续真实部署推理速度

### 12.4 动捕绝对位姿如何使用更合理
虽然状态分支直接输入动捕位姿合理，但要评估：

- 是否直接用全局坐标
- 还是做适当归一化或相对化变换
- 如何避免策略对场地固定坐标过度过拟合

### 12.5 sim-to-real 中最关键的误差来源
需要明确未来迁移时最可能出问题的是：

- 深度图噪声
- 遮挡
- 控制时延
- 速度跟踪误差
- 时间同步问题
- 车间最小距离安全约束

## 14. 当前建议的结论

当前最推荐的项目落地路线是：

> 在现有控制器和仿真系统基础上，采用“深度图视觉分支 + 自车状态分支 + 中间特征融合 + GRU 时序建模 + 共享策略 + CTDE + 几何结构 reward”的端到端高层协同强化学习方案，先在 Isaac Lab 中完成三车围捕静止目标训练，再逐步加入感知噪声、控制扰动和 sim-to-real 适配。

该路线的优点是：

- 符合老师提出的端到端方向
- 与现有系统接口匹配
- 能充分利用深度图与动捕状态
- 有较强工程可落地性
